Wallet Hierarchy Visual Guide Updated: April 18, 2026

Deterministic wallet architecture from mnemonic to private notes.

One mnemonic root now maps to the active Atho wallet model: a regular payment address for public transfers plus private bundle / X-address material for Platinum Shield receives. Older bond and stake references are historical background only.

Simple Mental Model

  • Mnemonic words Your recovery root. Same words, passphrase, and path rebuild the same wallet tree.
  • Two cryptographic branches Falcon handles identity and signing, while Kyber handles private payload encryption paths.
  • Two active address surfaces Regular payment address for public value, private bundle identity for Platinum Shield receive flows.
  • Recovery is two-stage Rebuild deterministic identity first, then rescan chain state to restore confirmed balances and notes.
Visual Map

One mnemonic root, multiple deterministic identities.

Mnemonic to identity path Mnemonic plus optional passphrase normalize into a deterministic PBKDF2-HMAC-SHA3-512 seed. That seed forks into a Falcon payment branch for public addresses and an isolated Kyber-backed branch for private bundle identity.
Mnemonic + Seed Math

Exact mnemonic sizing, checksum, and PBKDF2 behavior.

The wallet supports multiple mnemonic sizes, uses checksum bits derived from entropy, and turns the normalized mnemonic into a deterministic seed through PBKDF2-HMAC-SHA3-512.

Supported Mnemonic Sizes

12 words16 bytes entropy (128 bits)
24 words32 bytes entropy (256 bits)
48 words64 bytes entropy (512 bits)
checksum_bits = entropy_bits / 32
bitstream = entropy_bits + checksum_bits
words = bitstream / 11

Seed Derivation

seed = PBKDF2-HMAC-SHA3-512(
  password = normalized_mnemonic,
  salt = "atho-mnemonic-v1" + normalized_passphrase,
  iterations = 600000,
  dklen = 64
)
NFKD normalization Iteration guard: minimum 100000 Output: 64 bytes

Mnemonic Checksum Visual

Checksum summary Entropy contributes checksum bits at entropy_bits / 32, then the combined stream is sliced into 11-bit words. Supported sets remain 12, 24, and 48 words before deterministic expansion into the 64-byte wallet seed.

Falcon Branch Derivation

Falcon deterministic seed uses a dedicated domain-separated HKDF context so signing identity stays isolated from every other branch. 

context = "ATHO-HD-FALCON-V1"
info = "atho-mnemonic-v1|<network>|<account>|<role>|<index>"
falcon_seed_64 = HKDF-SHA3-512(seed, context, info, 64)
Deterministic Falcon keygen Role + network scoped

Kyber Branch Derivation

Kyber deterministic seed uses a distinct context and explicit level suffix, preventing Falcon and Kyber from silently coupling to the same derivation domain. 

context = "ATHO-HD-KYBER-V1"
info = "atho-mnemonic-v1|<network>|<account>|<role>|<index>|kyber=<level>"
kyber_seed_64 = HKDF-SHA3-512(seed, context, info, 64)
Independent branch domain Level-aware derivation info

Private Bundle Layout

Bundle byte order Each private bundle starts with a version byte, then appends Falcon public-key bytes and Kyber public-key bytes in fixed order. The public X-address is derived from Base56(SHA3-384(bundle_bytes)) with an X prefix.

Private Note Math

Commitment path Private note identity stays deterministic through domain-separated hashing, account indexing, and monotonic counters. Nullifiers and note commitments stay chain-bound so replay and cross-network reuse fail closed.

Spend-Safety Rules (Conceptual)

note_master = SHA3-256(
  "atho/private-note-master/v1" || network || falcon_pub || f || g || F || G
)

note_seed = SHA3-256(
  domain || network || note_master || account_u32_le || counter_u64_le
)
  • Nullifier uniqueness Rejects private replay and double-spend attempts.
  • Anchor and Merkle-path checks Spend proof must reference valid note-tree membership.
  • Signature and amount constraints Malformed payloads and invalid arithmetic are rejected fail closed.

Private Send Flow

Spend sequence Select mature notes, build the proof payload, validate anchors and nullifiers, then clear after confirmation policy is met. Fee policy stays simple: one floor, one minimum, no extra routing layer.

Recovery Flow

Restore sequence Rebuild deterministic identity first, then rescan chain state to reconstruct confirmed public balances, private notes, and mature spendability. Local cache and mempool hints are disposable; consensus-confirmed chain data is the source of truth.

Recovery Includes Two Layers

  • Deterministic identity rebuild Mnemonic, passphrase, derivation path, and network recreate Falcon identity, payment address, and private bundle identity.
  • Chain-state reconstruction Full sync or rescan restores confirmed UTXOs, private notes, and matured spendability state.

What Mnemonic Alone Does Not Restore

  • Unfinalized local mempool metadata Temporary local-only markers are not canonical if they never finalize on-chain.
  • UI cache artifacts Display cache is reconstructable, but it is not equivalent to consensus-confirmed balance truth.
