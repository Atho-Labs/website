Network

Built for safe throughput, not shortcuts.

As a Rust payment stack, Atho’s speed work is not about skipping checks. It is about validating on the canonical path, reusing exact safe results only when the context still matches, and keeping transport compact enough for real payment flow.

Performance without hidden consensus shortcuts.

The current strategy is straightforward: validate once on mempool admission, batch chainstate work, parallelize independent Falcon checks, and move less redundant data across the network.

  • Exact cache reuse only when bytes and rules still match
  • Batched UTXO reads and writes on the hot path
  • Parallel Falcon-512 verification for independent inputs
  • Compact relay and headers-first sync to cut wasted bandwidth
